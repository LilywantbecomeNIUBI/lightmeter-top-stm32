# 屏幕提示和错误码使用说明
## 先看哪个位置

开机后会先出现自检页面，大约 6 秒后进入正常测光页面。也可以长按编码器按键约 5 秒，再次进入自检页面。

正常测光页面底部有三段状态：

```text
L:80        D:0.42m        EV9.3
```

- `L:` 是光照值，单位近似为 lux。
- `D:` 是测距值，单位是 m。
- `EV` 是曝光值。
- 数字前面的 `~` 表示读数还在稳定，等一两秒再看。
- `--` 表示暂时没有有效读数。

## 自检页面提示

| 屏幕显示 | 给普通用户的意思 | 处理方法 |
|---|---|---|
| `ENC : OK` | 旋钮已经被识别 | 正常 |
| `ENC : TURN` | 还没检测到旋钮转动 | 转一下旋钮 |
| `MODE: OK` | 模式按键已经被识别 | 正常 |
| `MODE: PRESS` | 还没检测到模式按键 | 按一下旋钮按键 |
| `ISO : OK` | ISO 按键已经被识别 | 正常 |
| `ISO : PRESS` | 还没检测到 ISO 按键 | 按一下 ISO 按键 |
| `LUX : OK` | 光照传感器正常 | 正常 |
| `LUX : OFF` | 光照传感器没有读到 | 检查 VEML7700 模块、电源、I2C 接线 |
| `DIST: OK 42cm` | 测距模块正常，距离约 42 cm | 正常 |
| `DIST: WAIT` | 测距模块正在启动或等待第一笔数据 | 等一两秒 |
| `DIST:S02 42cm` | 测距模块回了数据，但认为这次距离不可靠 | 查下面的 `Sxx` 表，`42cm` 只能当参考 |
| `DIST:ID EACC H0` | 测距模块 ID 检查结果 | `EACC H0` 通常正常；其他值要检查模块型号、供电和 I2C 接线 |
| `DIST: BOOT` | 测距模块没有完成启动 | 重新上电；检查供电是否稳定 |
| `DIST: ID` | 读到的测距模块 ID 不对，或没读到 ID | 检查是不是 VL53L1X、I2C 线和焊接 |
| `DIST: CFG` | 测距模块配置失败 | 重新上电；若反复出现，检查模块或固件 |
| `DIST: INT` | 测距模块中断脚配置失败 | 检查 PC14/INT 接线，或先重新上电 |
| `DIST: CLR` | 清除测距模块中断失败 | 检查 I2C 接线和模块供电 |
| `DIST: START` | 测距启动失败 | 重新上电；检查测距模块 |
| `DIST: READ` | 读取测距结果失败 | 检查 I2C 接线、供电和模块是否松动 |
| `DIST: TRY` | 程序正在尝试恢复测距模块 | 等待；若一直不变，重新上电 |
| `DIST: OFF` | 当前没有启用测距 | 当前固件一般不会主动关闭测距；若出现，先重启 |

`H0/H1/H2/H3` 是程序底层的通信状态：`H0` 表示通信正常，`H1` 表示普通错误，`H2` 表示总线忙，`H3` 表示超时。

## 测距模块 `Sxx` 状态码

`Sxx` 来自 VL53L1X 测距芯片。只要看到 `Sxx`，就表示这次测距不能直接当作可靠距离使用。正常情况下，状态码 `S00` 不会显示成 `S00`，屏幕会显示 `DIST: OK` 或 `D:0.42m`。

| 代码 | 给普通用户的意思 | 常见原因 | 处理方法 |
|---|---|---|---|
| `S00` | 测距有效 | 正常读数 | 正常使用 |
| `S01` | 读数波动太大，不稳定 | 手抖、目标边缘、环境红外干扰、距离接近极限 | 对准平整目标，拿稳，缩短距离，等一两秒 |
| `S02` | 返回信号太弱 | 目标太远、太黑、太小、角度太斜 | 换白色/大一点的目标，靠近一些，垂直对准 |
| `S03` | 目标太近，低于可稳定测量范围 | 物体贴得太近 | 把目标移远，至少留出几厘米距离 |
| `S04` | 结果超出芯片能可靠判断的范围 | 距离接近极限，或反射条件异常 | 不要使用这次读数；靠近目标或换目标表面 |
| `S05` | 测距芯片硬件或激光发射部分异常 | 模块损坏、供电不稳、焊接问题 | 重新上电；仍出现就检查或更换测距模块 |
| `S06` | 这次读数没有完成防“绕回”检查 | 刚开始测距、远处高反光目标 | 等下一次读数；避开镜面、高反光远目标 |
| `S07` | 远处高反光目标导致距离“绕回” | 目标太远但很亮，例如反光牌、镜面 | 这次距离不可信；靠近目标或换非反光表面 |
| `S09` | 近处串光/盖板反光影响信号 | 传感器窗口脏、有透明盖板、近处强反光 | 清洁窗口，移除遮挡，避开近处反光物 |
| `S0A` | 启动后的同步提示，第一笔数据应忽略 | 刚开始测距 | 等下一次读数即可 |
| `S0B` | 多个目标的回波混在一起 | 传感器视野里有前后两个物体 | 只对准一个平整目标，避开复杂背景 |
| `S0C` | 能感觉到有目标，但信号太弱，报不出可靠距离 | 目标远、暗、小、倾斜 | 靠近目标，换更亮更大的目标 |
| `S0D` | 测距区域配置异常 | 当前固件通常不应出现，可能是模块状态异常或配置异常 | 重新上电；反复出现时检查固件和测距模块 |
| `SFF` | 没有更新到有效测距值，或没有检测到目标 | 刚启动、目标太远、传感器没对准目标 | 等一两秒；把目标放近并对准 |

当前固件的映射表会显示上表这些 `Sxx`。ST 官方资料里还定义了 `S08`、`S0E` 等状态，但按当前工程里的映射表，它们一般不会直接显示出来。

## 快速排查顺序

1. 先看 `LUX`：如果 `LUX : OFF`，光照传感器或 I2C 总线可能有问题，测光结果也不可靠。
2. 再看 `DIST`：如果是 `WAIT`，先等；如果是 `BOOT/ID/CFG/START/READ`，优先检查测距模块供电和 I2C 接线。
3. 如果是 `S02/S04/S07/S0C`，多数不是单片机坏了，而是目标太远、太暗、太小或太反光。
4. 如果是 `S05`，优先怀疑测距模块硬件、电源或焊接。
5. 处理后重新上电，再进入自检页面确认 `LUX : OK` 和 `DIST: OK xxcm`。
